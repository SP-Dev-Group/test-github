# The Base44-to-GitHub Static Migration Blueprint

This guide ensures that any React + Vite application exported from Base44 can be modified entirely within GitHub to run on static hosting (GitHub Pages) without requiring access to the internal Base44 SDK or backend servers.

---

### Step 1: Strip or Comment Out App-Level Dead Weight

Base44 templates ship with default Progressive Web App (PWA) configurations and hooks that break on static servers.

1. **Open `index.html**` in your root directory.
2. **Comment out the Web App Manifest link** to prevent a 404 HTML injection crash:
```html

```


3. **Update the Browser Tab Favicon Link** to point to your local repository asset instead of the dead external link:
```html
<link rel="icon" type="image/png" href="./SPFlyers-logo.png" />

```


*(Note: Ensure your custom logo image file is uploaded directly into the `public/` folder).*

### Step 2: Deploy the Core Mock Engine Interceptor

Since the actual `@base44/sdk` isn't accessible, we create a harmless local simulation file that satisfies the code's data requirements with static data.

1. Create a new file exactly at **`src/lib/base44-mock.jsx`**.
2. Populate it with this bulletproof mock fallback script:

```jsx
import React from 'react';

// 1. Simulates the Base44 database client and data hooks
export const createClient = () => ({
  entities: {
    useQuery: () => ({ data: [], isLoading: false, error: null }),
    useMutation: () => [async () => {}, { isLoading: false }],
    findOne: async () => ({}),
    findMany: async () => [],
    create: async () => ({}),
    update: async () => ({}),
    delete: async () => ({}),
  },
  auth: {
    me: async () => ({ id: 'mock-user', name: 'Test User' }),
    login: async () => ({}),
    logout: async () => ({}),
  },
  config: {
    logoUrl: '/test-github/SPFlyers-logo.png',
    theme: 'dark'
  }
});

// 2. Simulates global User Authentication states
export const useAuth = () => ({
  user: { id: 'mock-user', name: 'Test User', email: 'test@example.com' },
  isLoadingAuth: false,
  isLoadingPublicSettings: false,
  authError: null,
  navigateToLogin: () => console.log('Mock redirect to login'),
});

// 3. Simulates standard data query engine components
export const useQuery = () => ({
  data: {},
  isLoading: false,
  error: null,
});

export const queryClientInstance = {
  getQueryData: () => null,
  setQueryData: () => null,
};

// 4. Fallback Context Providers to wrap around your layout components
export const AuthProvider = ({ children }) => <>{children}</>;
export const QueryClientProvider = ({ children }) => <>{children}</>;
export const Base44Provider = ({ children }) => <>{children}</>;

export const useMutation = () => [async () => {}, { isLoading: false }];

const mockDefault = { createClient, useAuth, useQuery, AuthProvider, QueryClientProvider };
export default mockDefault;

```

### Step 3: Configure Vite to Hijack Imports

With the mock engine in place, we instruct the bundler to intercept any attempt to look for the missing external SDK and substitute it with our local mock file.

1. Open **`vite.config.js`** in your root directory.
2. Update or insert the `resolve.alias` mapping configuration to match this exact pattern:

```javascript
import react from '@vitejs/plugin-react'
import { defineConfig } from 'vite'
import path from 'path'

export default defineConfig({
  base: '/test-github/', // 👈 Ensure this matches your exact GitHub repository subfolder name
  logLevel: 'error',
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
      
      // INTERCEPT LOGIC: Force missing packages to use the local mock engine
      '@base44/sdk': path.resolve(__dirname, './src/lib/base44-mock.jsx'),
      '@/integrations/base44': path.resolve(__dirname, './src/lib/base44-mock.jsx'),
      '@/lib/AuthContext': path.resolve(__dirname, './src/lib/base44-mock.jsx'),
      '@/lib/query-client': path.resolve(__dirname, './src/lib/base44-mock.jsx'),
    },
  },
})

```

### Step 4: Neutralize Static Image Network Assets

If components hardcode references to online assets (like `https://base44.com/logo_v2.svg`), the browser will flag them with a `400 Bad Request` or a Cross-Origin (CORS) error.

1. Search the repository for **`logo_v2.svg`**.
2. Replace the source reference with your local subdirectory URL structure so it loads directly from your repository's compiled `public/` assets:
```html
<img src="https://base44.com/logo_v2.svg" />

<img src="/test-github/SPFlyers-logo.png" />

```

### The Golden Rule of Static Deployment Debugging

Whenever you push changes and encounter a blank screen or code behavior that seems "stuck in the past," remember that static entry elements like index headers and manifests are aggressively cached by browsers.

* Always force-bypass the browser cache using **`Ctrl + Shift + R`** on your Chromebook, or check the deployment via a fresh **Incognito Window (`Ctrl + Shift + N`)**.
